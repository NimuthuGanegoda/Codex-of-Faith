# Contributing to the Codex of Faith

First off, thank you for considering contributing to the Codex of Faith! It's people like you that make open-source such a fantastic place to learn, inspire, and create.

## How Can I Contribute?

### 1. Reporting Bugs & Issues
If you find a typo, a bug in the web application, or a factual inaccuracy in the religious texts, please open an issue in the GitHub repository. Provide as much detail as possible, including:
- What the issue is.
- Where it is located (file path or URL).
- Proposed corrections (with cited sources if factual).

### 2. Suggesting Enhancements
We are always looking to improve. You can suggest:
- New stories or parables.
- Summaries of additional belief systems.
- UI/UX improvements for the React frontend.

### 3. Submitting Pull Requests
1. Fork the repository.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. If you modified the web application, ensure `npm run build` completes without errors.
5. Commit your changes (`git commit -m 'Add some amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request.

## Code Style and Standards
- **Markdown:** Ensure all `.md` files are properly formatted and easy to read.
- **Web App:** Follow standard React/TypeScript conventions. Keep styling clean and within `App.css`.

Please remember to be respectful and considerate of all faiths and beliefs when contributing to this repository.
